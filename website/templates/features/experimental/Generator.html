@Generator was introduced as experimental feature in lombok v0.18.27.

<@f.experimental>
  • Unstable feature. This feature is not tested extensively and needs more experiment.
<@f.overview>

Generator, also known as Sequence is a form of iterator which can be yielded and resumed later. Their context(variable declarations) will be saved for future resumption.

By default method annotated with @Generator will not be run immediately on called. Instead the method returns Iterator(or Iterable). By calling next method of iterator it will be executed until next yieldThis statement is met, which is the next value of iterator, or with yieldAll statement, delegates to another iterable object. The next or hasNext method will resume iterator and buffer next yielded value if there is no buffered value. Additionally the next method will return buffered value and remove it.

All statements containing control flow change statements including yieldThis, yieldAll, return, break, continue and throw will be transformed.
All local variables will be captured as a field. Captured objects can be garbage collected just like local variables.

<@f.snippets name="experimental/Generator" /> <@f.confKeys>
lombok.generator.flagUsage = [warning | error] (default: not set)
Lombok will flag any usage of @Generator as a warning or error if configured.
<@f.smallPrint>

When using yieldAll statement, using primitive type array is impossible to use due to limitation.
